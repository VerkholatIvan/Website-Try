The most basic code we can write with Flask is this:

from flask import Flask

app = Flask(__name__)

@app.route("/")
def hello_world():
    return "<p>Hello, World!</p>"

This little code will just create a basic page with one message.

However, how to see the result, since it is not the HTML file. Well, all you need to do is to write down this command in the terminal:

>> flask --app hello run

Using it you will have a working link. But do not forget to end the process by the Ctrl+C buttons, because while working, your computer becomes the server to your website, which makes you an easy target for hackers.

There's also a debugging mode, which allows one to see the errors if there are any.

To call it, all you need to do, is just add the --debug option:

>> flask --app hello --debug run

Regardless, let's discuss our little code.

Firts of all, we have imported Flask.

Then we need to create an instance of this class. The __name__ is a useful and convenient shortcut for that and which is suitable for most cases.

When using route() we tell which URL the page will have.

So, basically we can have several routes with their own names:

@app.route('/')
def index():
return 'Hello, user'

@app.route('/weather')
def weather():
    return 'Today's weather is...'

But before ending this section let's discuss what can we use Flask for and why people don't want to use or on the contrary, would really like to.

Flask can look difficult because of the unusual coding way. Anyway, there are some advantages of it:

• Flexible

• Easy to negotiate

• Lightweight

Still, cons such as lack of tools, more difficult to get familiar with a larger Flask apps and maintenance costs, usually discourages companies from using it. On the contrary, it still can be really useful working with small projects.
